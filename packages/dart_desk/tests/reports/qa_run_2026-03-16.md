## Key Findings

### Blocking Issues
1. **Version not auto-selected from DocumentRoute** — When a document is selected via the document list, the route pushes `/test_all_fields/{id}` but the editor requires a version ID in the URL (`/test_all_fields/{id}/{versionId}`) to load data. Without auto-selecting the latest version, the editor shows default/empty fields instead of seed data. This blocks tests 04-08.

2. **Marionette `enter_text` doesn't trigger `onChanged`** — On Flutter web, marionette's text injection updates the controller value but doesn't fire the `onChanged` callback on `ShadInputFormField`. This blocks search and text editing tests.

3. **Hot reload breaks ZenRouter state** — Hot reload causes the coordinator to lose its navigation state, resulting in a blank screen. Only full restart recovers. This slows fix-and-test cycles significantly.

### Recommendations
1. **Auto-select latest version** when a `DocumentRoute` is pushed (no version segment). In `setRouteParams`, if `documentId` is set but `versionId` is null, fetch versions and auto-select the latest.
2. **Add ValueKeys** to document tiles and form fields for more reliable marionette targeting.
3. **Investigate marionette web text injection** — may need to dispatch `TextInputClient` events rather than setting controller text directly.
