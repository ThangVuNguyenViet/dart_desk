# DocumentTypeSpec Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Introduce `DocumentTypeSpec<T>` to break the circular dependency between `data_models` and `your_app` by moving the preview builder out of data_models and into cms_app.

**Architecture:** A new `DocumentTypeSpec<T>` class is added to `dart_desk_annotation`. The generator emits a `*TypeSpec` variable instead of a full `*DocumentType`. The cms_app calls `.build(builder: ...)` to produce the final `DocumentType`, supplying the screen widget import itself.

**Tech Stack:** Dart, Flutter, source_gen/build_runner (code generation), dart_desk_annotation, dart_desk_generator

- [ ] **Step 1: Add `DocumentTypeSpec<T>` class**

Open `packages/dart_desk_annotation/lib/src/config.dart` and append after the closing `}` of `DocumentType`:

```dart
/// A partial document type definition without a preview builder.
///
/// Generated by dart_desk_generator for each @CmsConfig class.
/// Call [build] in your cms_app to produce a full [DocumentType]
/// by supplying the preview builder there.
///
/// Example:
/// ```dart
/// final myDocumentType = myTypeSpec.build(
///   builder: (data) => MyScreen(config: MyConfigMapper.fromMap(data)),
/// );
/// ```
class DocumentTypeSpec<T extends Serializable<dynamic>> {
  /// Unique identifier for this document type
  final String name;

  /// Display title for this document type
  final String title;

  /// Description of this document type
  final String description;

  /// List of fields for this document type
  final List<CmsField> fields;

  /// Optional default instance of T
  final T? defaultValue;

  const DocumentTypeSpec({
    required this.name,
    required this.title,
    required this.description,
    required this.fields,
    this.defaultValue,
  });

  /// Produces a full [DocumentType] by supplying a preview [builder].
  ///
  /// The [builder] receives the raw CMS data map and must return a Widget.
  /// This is required — it is a compile error to call [build] without it.
  DocumentType<T> build({
    required Widget Function(Map<String, dynamic> data) builder,
  }) {
    return DocumentType<T>(
      name: name,
      title: title,
      description: description,
      fields: fields,
      builder: builder,
      defaultValue: defaultValue,
    );
  }
}
```
