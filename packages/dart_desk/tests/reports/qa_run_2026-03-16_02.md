## Context
This run validates the live preview feature implemented in this session:
- `_testAllFieldsBuilder` now renders actual field data with `preview:{field}:` prefixes
- Shared `editedData` signal on `CmsDocumentViewModel` links editor and preview
- Auto-version selection when opening a document (previously `versionId` was null)
- Preview reads from `editedData` first, falls back to saved version data

## Bug Found & Fixed During Testing
**Auto-version selection**: When opening a document via `DocumentRoute`, `versionId` was null because `DocumentRoute` doesn't include a version. Added `_autoSelectLatestVersion()` in `CmsViewModel.setRouteParams` to fetch and select the latest version automatically.

**Assertion error in editor**: The `addPostFrameCallback` in `document_editor.dart` caused `_owner != null` and `_elements.contains(element)` assertion failures. Removed the callbacks since `_autoSelectLatestVersion` now pre-populates `editedData` directly.
